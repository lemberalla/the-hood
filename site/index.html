What ships in preview

The preview is intentionally local and evidence-first. It is not a hosted agent service, scheduler, or polished app platform.

Runtime

Loop control lives locally

The runtime owns approvals, artifacts, validation evidence, review routing, and terminal state.

Roles

Provider-neutral work lanes

Assign Codex, Claude Code, ChatGPT Web, or stub providers to planner, implementer, QA, verifier, and critic roles.

Boundary

No hidden cloud loop

The preview does not include hosted execution, timer schedules, production API adapters, or private artifact publishing.

The loop path

A goal advances only through runtime-owned state. Provider output is input, not authority.

1Goal

Define the software outcome, repo, role map, and iteration budget.

2Delegate

Send bounded directives to configured role owners.

3Capture

Record command logs, git evidence, artifacts, and provider responses.

4Verify

Run deterministic checks and independent review lanes.

5Stop

Complete, ask for approval, fail closed, or stop at a budget cap.
