Runtime-owned progress

Goal Loops

The preview separates goals, loops, and schedules so public claims stay honest.

Goal

A goal is a concrete software objective with a verifiable stop condition.

thehood goal "Prepare release metadata" --repo . --max-iterations 5

Loop

A loop advances a run while the local runtime process is active. It stops at terminal state, manual approval, no progress, or a budget cap.

PlanDirective

Runtime sends bounded instructions.

ActRole owner

The configured provider returns schema-bound output.

RecordEvidence

The runtime writes artifacts and command logs.

ReviewLanes

QA, verifier, and critic paths are separated.

StopState

Completion, gate, failure, or budget stop.

Schedule

Schedules are future work. The public preview does not include timer loops, background daemons, hosted queues, or overnight automation.

Source docs: `docs/GOAL_LOOP_SCHEDULE.md` and `docs/RUNTIME_LOOP.md`.
