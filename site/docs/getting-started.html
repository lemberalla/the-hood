Local preview

Getting Started

The public preview is designed to work from a local checkout first. After npm publishing, the same commands should work through the `thehood` binary.

Build

npm install
npm run build
npm run smoke:runtime
npm run smoke:mcp

Initialize A Repo

node dist/cli/main.js init --repo .
node dist/cli/main.js doctor --repo .
node dist/cli/main.js roster --repo .

Run The Synthetic Demo

The stub demo proves loop mechanics without external providers.

DEMO_REPO="$(mktemp -d)"
node dist/cli/main.js init --repo "$DEMO_REPO"
node dist/cli/main.js approvals policy set mode autopilot --repo "$DEMO_REPO"
cat > "$DEMO_REPO/package.json" <<'JSON'
{
  "scripts": {
    "typecheck": "node -e \"process.stdout.write('demo validation ok\\n')\""
  }
}
JSON
node dist/cli/main.js goal "Synthetic demo: prove a governed local loop can complete" \
  --repo "$DEMO_REPO" \
  --orchestrator stub:orchestrator \
  --implementer stub:implementer \
  --qa stub:qa \
  --verifier stub:verifier \
  --critic stub:critic \
  --max-iterations 5 \
  --json

Expected Result

  • The run reaches `completed`.
  • The verifier approves runtime evidence.
  • The validation summary reports zero failed commands.
  • No external model, browser bridge, API provider, or MCP connector is used.
Source docs: `docs/DEMO.md` and `examples/stub-demo/README.md`.
