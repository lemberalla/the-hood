Provider-neutral roles

Provider Matrix

Provider choice changes who supplies suggestions. It does not change runtime authority, approvals, role separation, or verification gates.

ProviderStatusPreview use
`stub`ImplementedDeterministic local smoke tests and synthetic demos. No external calls.
`codex-cli`ImplementedLocal Codex command adapter with schema-bound responses.
`claude-code`ImplementedLocal Claude Code command adapter for selected roles.
`chatgpt-web`ExperimentalUser-authenticated ChatGPT Web bridge when configured locally.

`openai-api`FutureConfigured metadata exists, but production API adapter wiring is not included.
`anthropic-api`FutureConfigured metadata exists, but production API adapter wiring is not included.

Role Policy

  • Implementer and verifier must not be the same agent for the same task.
  • Verifier and critic roles do not receive edit tools.
  • Provider adapters normalize responses into TheHood schemas.
  • Provider adapters must not bypass runtime permissions or transfer manifests.
Source docs: `docs/PROVIDER_MATRIX.md` and `docs/ROLE_CONTRACTS.md`.
