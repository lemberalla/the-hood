Contributing

Public-preview contributions should preserve the runtime invariants and keep provider-specific behavior behind adapters.

Local Checks

npm run typecheck
npm run build
npm run smoke:mcp
npm run smoke:codex-config
npm run smoke:runtime

Pull Requests

  • Explain runtime, provider, approval, filesystem, MCP, or verification behavior changes.
  • Classify protected test, fixture, snapshot, or eval changes.
  • Keep examples synthetic.
  • Do not commit secrets, local `.thehood/` state, browser profiles, provider logs, or private run transcripts.

Provider Work

Provider adapters should normalize external behavior into TheHood schemas. They should not own approval state, bypass transfer manifests, or grant tools to roles that should remain read-only.

Source docs: `CONTRIBUTING.md` and `docs/CONTRIBUTOR_GUIDE.md`.
