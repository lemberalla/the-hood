Trust boundary

Security

TheHood keeps authority in the runtime. Provider sessions may help reason, but they do not own permissions, artifacts, approval state, or final verification.

Local

State stays on disk

Run records, artifacts, approval events, and local config live under the repo-local TheHood state directory.

Explicit

Transfers are gated

Before local repo context leaves the machine for browser or API providers, TheHood writes a transfer manifest.

Forbidden

No private artifacts

Do not publish `.thehood/`, browser profiles, provider transcripts, local env files, or real run logs.

Enforced

  • Models suggest; runtime enforces.
  • Implementer and verifier separation is mandatory.
  • Verifier and critic roles do not receive edit tools.
  • Protected test, fixture, snapshot, and eval changes require explicit classification.
  • Provider adapters must not bypass runtime permissions.
Source docs: `docs/SECURITY_AND_PRIVACY.md`, `PRIVACY.md`, and `docs/TRUST_MODEL.md`.
