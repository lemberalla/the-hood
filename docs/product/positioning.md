# Positioning

TheHood gives Codex a governed runtime plus an optional premium strategic brain.

The runtime controls execution. Pro improves judgment when the work is ambiguous, high-value, reputational, or needs reconciliation.

## Core Claim

TheHood is not a hidden bridge that lets Codex silently spend Pro reasoning. It is a local agent runtime that makes premium reasoning visible, policy-driven, and auditable.

## Product Shape

- Runtime conductor: deterministic loop control, role separation, approvals, evidence, artifacts, and verification gates.
- Codex-first execution: Codex remains the default control and implementation surface.
- Visible Pro escalation: Pro is used deliberately for strategic planning, product judgment, reconciliation, critique, and high-reputation review.
- Connector fallback: when direct Pro calls are blocked by Codex or tenant host policy, ChatGPT MCP connector mode is the safe handoff path.

## Tradeoff

The original pitch, "Codex can use ChatGPT Pro," is simple and compelling. It is also too easy to misread as Pro being the hidden conductor.

The stronger pitch is more precise:

- The runtime executes and enforces.
- Pro advises and judges.
- Users can see when Pro is used, why it is recommended, and whether it was automatic.

This is slightly less viral than the original pitch, but it is more trustworthy and easier to defend for serious software work.

## User-Facing Line

Use TheHood when you want Codex to run through a governed local runtime, with ChatGPT Pro available as a visible strategic reviewer for decisions that deserve premium judgment.
